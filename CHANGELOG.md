# 更新日志

## 1.7.6（2026-06-15）

### 新功能
- 新增 AI 电脑管家工具（`ai-butler/`）：本地多 Agent 任务编排系统，支持代码、文件、系统、浏览器四类内置 Agent，带 Web UI 控制台。
- 任务调度引擎支持优先级队列、并发控制和智能 Agent 匹配。

### 改进
- 修复 Webview 消息组件中因编码问题产生的乱码字符。
- 消息复制按钮增加 `instanceof HTMLElement` 类型安全检查。

### 修复
- 修复 `messages.ts` 中注释和分隔线的 Unicode 编码乱码问题。
- 新增任务变更编辑计数、剩余计数和确认输入的 i18n 翻译键。

## 1.7.5（2026-06-15）

### 新功能
- 输入框支持项目文件快捷引用：输入 `#` 或点击左下角 `+` 可从项目文件树中选择文件，并以内联文件 token 的形式插入消息。
- 文件引用 token 支持悬停查看完整路径，发送时会转换为完整文件路径，便于模型准确读取目标文件。

### 改进
- 优化文件选择器的大项目体验，文件树不再因为条目过多遗漏正常文件夹，搜索结果和树形浏览更加一致。
- 改进输入框文件 token 的光标定位、选中文本颜色和叠字问题，使普通文本选择与文件引用显示更稳定。
- 优化 Webview 性能路径，降低长推理、edit/diff、workflow 和历史回放带来的滚动与交互卡顿。
- 极速推理模式加入更简洁的显示风格，减少彩色强调和冗余视觉元素，提高长输出下的可读性与流畅度。
- 历史记录恢复会话时同步输入历史，上键可继续调出历史输入消息。
- 历史记录中的 Thought/思考内容改用历史专用展示逻辑，实时运行时保护长草稿，历史回放时仍可展开查看已保存过程。

### 修复
- 修复 MiMo 多轮工具调用中 `reasoning_content` 被压缩后继续发送，可能导致 400 请求体格式错误的问题。
- 修复无限轮次预算在错误摘要中显示为 `9007199254740991` 的问题，改为显示 unlimited budget。
- 修复只读或无本轮代码修改时仍显示旧 Diff 卡片的问题。
- 修复 Tool Edit/Diff 统计把少量改动显示成数千行增删的问题。
- 增强需要工具证据的问题路由，减少未检查项目/接口/来源就直接回答或提前结束的情况。
- 修复思考内容隐藏提示暴露内部协议字段的问题，不再向用户展示 `reasoning_content` 等实现细节。

## 1.7.4（2026-06-13）

### 改进
- 修复 VS Code Remote WSL 场景下，Windows 系统代理或 VS Code 代理注入 `PROXY 127.0.0.1:7890` 时导致 MiMo API 请求失败的问题。
- 为 WSL loopback 代理场景加入直接 socket 请求路径，在需要绕开错误代理注入时不再依赖 VS Code/Node 的 HTTP 请求代理层。
- 保留显式可用代理配置的正常支持，同时对 WSL 中不可达的本地代理注入进行自动忽略或改写处理。
- Bash/`execute_command` 工具卡片在命令完成后默认保持 Hide/折叠，长命令输入和输出需要用户手动点击 Show 查看。
- 打包时排除测试生成的 AI 新闻总结文件，避免无关文件进入 VSIX。

### 修复
- 修复 WSL 中开启 Windows 系统代理后出现 `Failed to establish a socket connection to proxies: PROXY 127.0.0.1:7890` 的问题。
- 修复早期 WSL 代理绕过实现等待 SSE 连接自然关闭才完成，导致任务耗时异常变长的问题；现在会增量解析 SSE，并在收到 `data: [DONE]` 后立即结束。
- 修复同版本反复测试时容易安装到旧 VSIX 的混淆问题，将本次最终代理修复统一整理为 1.7.4 发布。
