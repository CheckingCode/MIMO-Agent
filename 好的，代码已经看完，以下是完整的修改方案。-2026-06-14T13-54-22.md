## 📋 文件引用功能 — 修改方案

### 整体架构

```
用户输入 # → 前端检测 → 后端搜索文件 → 弹出选择列表 → 插入彩色标签 → 发送时解析为完整路径
```

### 上次失败的教训

1. **Enter 键被拦截** — `fileSearchState.active` 为 true 时，Enter 键被文件搜索吞掉，导致正常消息发不出去
2. **没有防抖** — 每敲一个字都触发后端搜索
3. **条件过严** — `#` 必须在行首或空格后才触发，用户输入"分析#"无法触发

### 4. 前端核心：`input.ts`

**数据结构**：
```typescript
// InputArea 上新增属性
attachedFiles: Array<{ name: string; relativePath: string; fullPath: string }>
fileSearchState: { active: boolean; query: string; triggerPos: number; selectedIndex: number }
searchDebounceTimer: ReturnType<typeof setTimeout> | null
```

**新增方法**：

| 方法 | 职责 |
|------|------|
| `initFileSearch()` | 在 mount() 中调用，创建弹窗 DOM，监听 `fileSearchResults` 事件 |
| `handleFileTrigger(input)` | 在 input 事件中调用，检测 `#` 并触发搜索 |
| `renderFileSearchResults(results)` | 渲染搜索结果到弹窗 |
| `closeFileSearch()` | 关闭弹窗并重置状态 |
| `insertFileReference(fullPath, name, relativePath)` | 插入文件标记到 textarea |
| `renderFileTags()` | 在 textarea 上方渲染彩色文件标签 |
| `removeFileReference(index)` | 移除指定文件引用 |
| `resolveFileMarkers(text)` | 发送前将 `@[file:路径]` 解析为 `[File: 路径]` |

**关键修复点（对比上次）**：

1. **防抖**：`handleFileTrigger` 中用 `setTimeout` 200ms 防抖，避免频繁搜索
2. **Enter 键安全**：在 keydown 处理器最开头，先检查光标是否还在 `#query` 范围内，不在则自动关闭搜索状态，**之后再走正常的 Enter→doSend 逻辑**
3. **触发条件放宽**：移除"必须在行首或空格后"的限制，`#` 出现在任何位置都能触发
4. **弹窗关闭时机**：`doSend()` 开头先调用 `closeFileSearch()`

**修改的现有方法**：

- `mount()`：末尾调用 `this.initFileSearch()`
- `doSend()`：开头调用 `this.closeFileSearch()`，发送时调用 `this.resolveFileMarkers(text)`
- keydown 处理器：最开头插入文件搜索键盘导航（↑↓ 选择、Enter 确认、Esc 关闭），但 **Enter 确认后 return，不走 doSend**；光标离开 `#query` 范围则关闭搜索

### 6. 用户交互流程

```
1. 用户输入 "分析#input"
                          ↓
2. input 事件 → handleFileTrigger → 检测到 # → 200ms 防抖
                          ↓
3. vscode.searchFiles("input") → 后端搜索 → 返回匹配文件
                          ↓
4. fileSearchResults 事件 → renderFileSearchResults → 弹窗显示
                          ↓
5. 用户 ↑↓ 选择 → Enter 确认 → insertFileReference
                          ↓
6. textarea 显示 "分析@[file:/path/to/input.ts]"
   输入框上方显示标签: [input.ts ×]
                          ↓
7. 用户输入其他内容 → 点击发送
                          ↓
8. doSend → resolveFileMarkers → vscode.send("分析 [File: /path/to/input.ts]")
```
