## v1.3.0 核心变更：架构重构

### Before → After

```
v1.2.0 (单文件架构):                v1.3.0 (组件化架构):
src/webview/                         src/webview/
├── chatProvider.ts (630 行)          ├── chatProvider.ts (270 行)
│   ├── 250 行 CSS 内联              ├── styles.css (独立 CSS 文件)
│   ├── 90 行 HTML 内联              ├── main.ts (入口 + 消息路由)
│   └── 全部逻辑混杂                 ├── core/
└── app.js (567 行 vanilla JS)       │   ├── store.ts (集中式状态管理)
    └── var 全局变量 + IIFE           │   ├── bus.ts (事件总线)
                                     │   └── vscode.ts (API 封装)
                                     ├── utils/
                                     │   └── dom.ts (DOM 工具)
                                     └── components/
                                         ├── header.ts
                                         ├── messages.ts
                                         ├── input.ts
                                         ├── panels.ts
                                         ├── commandPalette.ts
                                         └── imageUpload.ts
```

### 架构对比

| 维度 | v1.2.0 | v1.3.0 | Claude Code |
|------|--------|--------|-------------|
| **模块系统** | 单 IIFE + var 全局 | ES modules + esbuild 打包 | Webpack + React |
| **状态管理** | 闭包变量 | 集中式 Store + 事件订阅 | React hooks + Zustand |
| **组件通信** | 直接函数调用 | 事件总线解耦 | Props + Context |
| **VSCode API** | 散布各处 | 统一封装 (vscode.ts) | 统一 API 层 |
| **CSS** | 250 行内联在 TS 模板 | 独立 styles.css | CSS Modules / styled |
| **单文件行数** | 567 (app.js) | 30-120 (每个组件) | 50-150 (每个组件) |
| **构建** | `cp app.js` | esbuild (6ms) | Webpack / esbuild |
| **可维护性** | 🔴 极低 | 🟢 高 | 🟢 高 |
| **可测试性** | 🔴 无法测试 | 🟡 可独立测试组件 | 🟢 完整测试套件 |
| **新功能扩展** | 🔴 需理解整个文件 | 🟢 添加新组件即可 | 🟢 添加新组件 |
