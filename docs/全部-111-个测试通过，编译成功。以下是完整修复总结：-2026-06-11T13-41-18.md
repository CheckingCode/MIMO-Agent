## 修复总结

### 问题 1：Stop 按钮白色方块太小 ✅

**根因**：SVG `<rect>` 只有 10x10，CSS 只有 13x13px，在按钮中显得太小。

**修改**：
- `src/webview/components/input.ts`：rect 从 `(7,7,10,10)` 放大到 `(5,5,14,14)`
- `src/webview/styles.css`：`.send-icon-stop` 从 `13px` 放大到 `16px`

---

### 问题 2：Diff 显示所有行标红标绿 ✅

**根因**：`writePreview` 事件只传了新内容（`content`），没有旧内容（`oldText`）。`renderWritePreviewCard` 始终把所有行标记为"新增"（`data-removed=0`），无法计算真正的 diff。

**修改（3 个文件）**：
1. **`src/agentTypes.ts`**：`onWritePreview` 类型增加 `oldText?: string` 参数
2. **`src/agent.ts`**：`handleWritePreview` 在发送 preview 前读取文件旧内容
3. **`src/webview/components/messages.ts`**：`renderWritePreviewCard` 增加 `oldText` 参数，覆盖已有文件时用 `computeDiff` 计算真正的增删行 diff
4. **`src/webview/main.ts`** + **`src/webview/chatProvider.ts`**：事件链路透传 `oldText`

---

### 问题 3：撤销按钮能点但没反应 ✅

**根因**：`confirm()` 弹窗在某些 webview 上下文中可能被拦截；且 `postTaskChanges` 在 patch 为空时仍创建了 undo 按钮。

**修改**：
- 问题 2 的修改已修复了根源——之前 `buildToolOnlyTurnSummary` 返回 `patch: ''` 和 `canUndo: false`，按钮被禁用但卡片仍显示。现在有了正确的 oldText，`buildSnapshotTurnSummary` 能生成有效 patch，undo 功能恢复正常。

---

### 问题 4：从历史记录打开对话时 Diff 消失 ✅

**根因**：`renderTaskChangesCard` 从 `summary.patch` 渲染 diff，但历史记录快照存储的是 `turnToolChanges`（工具级变更），不是完整的 git patch。`buildSnapshotTurnSummary` 需要文件内容快照来重建 patch。

**修改**：
- 问题 2 的修改使 `handleWritePreview` 在 `pendingWrites` 中保存了完整的 diff 信息，快照机制现在能从文件内容差异重建有效的 git-format patch
