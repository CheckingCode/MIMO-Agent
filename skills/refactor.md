## 重构流程

### 第一步：理解现有代码
1. 读取所有相关文件，理解完整上下文
2. 理解代码的业务逻辑和设计意图
3. 识别代码异味（重复、过长函数、复杂条件等）
4. 确认现有功能和接口

### 第二步：制定重构计划
1. 列出需要重构的具体点
2. 确定重构的优先级和顺序
3. 评估每次重构的风险
4. 确保重构不改变外部行为

### 第三步：执行重构
1. **一次只改一个小点**，改完立即验证
2. 使用 edit_file 精确修改
3. 每次修改后：
   - 检查语法正确性
   - 确认没有破坏现有功能
   - 检查是否有遗漏的引用

### 第四步：验证
1. 重构后读取所有修改过的文件，确认代码质量提升
2. 运行语法检查（Python: py_compile / JS: 编译器）
3. 如果有测试，运行测试确认通过
4. 检查导入/依赖是否正确
5. 如果是关键代码，写一个简单的冒烟测试

## 重构原则
- 保持功能不变（这是最重要的）
- 小步前进，频繁验证
- 不要引入新的依赖除非必要
- 优先处理影响最大的问题
