## 调试流程

### 第一步：收集信息
1. 仔细分析错误信息、堆栈跟踪、日志输出
2. 定位错误发生的文件和行号
3. 读取相关文件的完整内容
4. 检查最近的修改（如果有 git，用 git diff 查看）

### 第二步：分析根因
1. 不要只看表面错误，追踪完整的调用链
2. 检查变量的值和类型是否正确
3. 检查数据流：输入 → 处理 → 输出
4. 确认依赖版本和环境配置
5. 查找类似问题的已知解决方案

### 第三步：制定修复方案
1. 提出可能的修复方案（至少2种）
2. 评估每种方案的风险和副作用
3. 选择最安全、最彻底的方案
4. 如果需要，先在小范围测试

### 第四步：执行修复
1. 使用 edit_file 精确修改，不要重写整个文件
2. 每次只改一个点，方便定位问题
3. 修改后立即验证语法正确性

### 第五步：验证修复
1. 重新运行出错的操作，确认错误消失
2. 检查修复是否引入了新问题
3. 如果有测试，运行测试套件
4. 如果是运行时错误，尝试复现原始场景

### 第六步：防止复发
1. 如果是常见错误模式，添加防御性代码
2. 改进错误提示信息
3. 如果合适，添加相关的测试用例

## 重要原则
- 不要猜测，要有证据
- 不要只修复表面症状，要找到根因
- 修复后必须验证，不能假设"应该可以了"
- 如果问题复杂，分步骤处理，每步都验证
