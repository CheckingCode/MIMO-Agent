## 文档流程

### 第一步：分析代码
1. 读取源文件，理解整体结构
2. 识别所有公开接口（函数、类、方法）
3. 理解参数类型、返回值、异常
4. 查找已有的文档，保持一致风格

### 第二步：生成文档
为每个接口生成：
1. **用途说明**：一句话描述功能
2. **参数说明**：类型、含义、默认值、约束
3. **返回值说明**：类型、含义
4. **异常说明**：可能抛出的异常
5. **使用示例**：简单实用的示例代码
6. **注意事项**：边界条件、性能考虑

### 第三步：写入文档
支持的格式：
- Python docstring（Google/NumPy 风格）
- JavaScript/TypeScript JSDoc
- Markdown README
- API 文档（OpenAPI/Swagger）
- 中文注释

### 第四步：验证
1. 检查文档格式是否正确
2. 确认示例代码可以运行
3. 检查是否有遗漏的接口
4. 确认与代码实际行为一致

## 原则
- 文档要简洁准确，不要废话
- 示例代码必须可运行
- 保持与项目现有文档风格一致
- 优先文档化复杂和关键的代码
