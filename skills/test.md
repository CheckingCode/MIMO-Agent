## 测试流程

### 第一步：分析代码
1. 读取源文件，理解所有函数/类的接口
2. 识别测试框架（pytest / unittest / Jest / Mocha 等）
3. 检查是否已有测试文件，避免重复
4. 分析依赖关系，确定需要 mock 的部分

### 第二步：设计测试用例
为每个函数/方法设计测试：
1. **正常路径**：典型输入，期望输出
2. **边界条件**：空值、零值、最大值、最小值
3. **错误处理**：异常输入、类型错误、缺失参数
4. **集成场景**：多函数协作的场景

### 第三步：编写测试
1. 使用框架约定的命名和结构
2. 测试名称清晰说明测试目的
3. 每个测试只验证一个行为
4. 包含必要的 setup/teardown
5. 使用 mock 隔离外部依赖

### 第四步：运行测试
1. 运行整个测试套件
2. 确认所有测试通过
3. 如果有失败，分析原因并修复
4. 检查代码覆盖率（如果工具支持）

### 第五步：验证
1. 确认测试覆盖了所有关键路径
2. 确认测试本身是可靠的（不是假阳性）
3. 如果发现代码 bug，报告给用户
